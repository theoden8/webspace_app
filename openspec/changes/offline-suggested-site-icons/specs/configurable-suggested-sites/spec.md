## MODIFIED Requirements

### Requirement: SUGGEST-001 - Flavor-Dependent Defaults

The default suggested sites list SHALL be identical across all build flavors. Because the suggestions list renders icons through the offline read path (SUGGEST-007) backed by committed bundled assets, shipping the curated defaults no longer causes third-party network contact, so the `fdroid` flavor SHALL ship the same curated defaults as `fmain` and `fdebug`.

#### Scenario: All flavors share curated defaults

**Given** the app is built with any flavor (`fdroid`, `fmain`, or `fdebug`)
**When** the user opens the "Add new site" screen for the first time
**Then** the same curated default list of suggested sites is shown (Duck.ai, Claude, ChatGPT, etc.)

#### Scenario: Empty state only after user removes all suggestions

**Given** the user has removed every suggested site
**When** the user opens the "Add new site" screen
**Then** an empty state message says "No suggested sites. Tap + to add some."

## ADDED Requirements

### Requirement: SUGGEST-007 - Suggestions List Makes No Network Requests

Rendering the suggested sites list SHALL make zero outbound icon requests. Each tile SHALL resolve its icon from local sources only: a committed bundled asset when one exists for the host, otherwise a locally-rendered monogram. The list SHALL never contact the outbound HTTP factory. Network fetching SHALL occur only when a suggestion or custom site is added (ICON-010), via the normal online icon widget on the main grid, not while viewing the suggestions list.

#### Scenario: Opening the suggestions list contacts no third party

**Given** a recording `OutboundHttpFactory` is installed
**When** the "Add new site" screen renders the curated suggested sites
**Then** the factory is never asked for a client
**And** no request reaches DuckDuckGo, Google, or any site favicon endpoint

#### Scenario: Suggestion without a bundled asset renders a monogram

**Given** a curated suggestion whose host is not in `kBundledIconHosts`
**When** the suggestions list renders its tile
**Then** a host-colored monogram is shown
**And** no network request is issued

#### Scenario: Bundled hosts are a subset of the curated suggestions

**Given** the bundled hosts (`kBundledIconHosts`)
**When** each is matched against the curated suggestion hosts
**Then** every bundled host corresponds to a curated suggestion
**And** every bundled host has a committed, loadable asset under `assets/suggested_icons/`

### Requirement: SUGGEST-008 - Bundled Suggestion Icons Are Committed, Not Built

The curated suggestion icons SHALL be committed assets shipped in the APK, regenerated by a developer-run script and never fetched during `flutter build`, so the build stays reproducible and the build server makes no third-party contact.

#### Scenario: Build performs no icon fetch

**Given** a clean checkout with no network access during the build
**When** the app is built with `flutter build apk --flavor fdroid`
**Then** the build succeeds using the committed bundled icon assets
**And** no favicon request is issued by any build step

#### Scenario: Regeneration is a manual runner

**Given** the suggestion-icon regeneration script under `scripts/`
**When** a developer runs it
**Then** it fetches current favicons for the curated suggestions and writes the asset files for committing
**And** it is not invoked by the Flutter or Gradle build
